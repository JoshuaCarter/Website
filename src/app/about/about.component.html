About This Website

This is the first website I've built. It is currently a work in progress and I'm still learning a lot as I go along. You can see the source-code via the github link in the nav menu.

I've made an effort to use modern tools and practices. The following technologies were used in this website's construction.

Languages:

  • HTML
    Also with Angular directives and bindings.
  • CSS
    From scratch, i.e. no bootstrap or similar css lib.
  • Typescript
    For the frontend with Angular (and because it's familier to me and my C# background).
  • PHP
    For the backend with MySQL.
  • MySQL
    Using the built-in mysqli lib in php.
  • JSON
    For config files and comms between frontend and backend.

Tools:

  • VSCode
    My chosen IDE.
  • Angular 4
    Chosen framework for responsive front-loaded design.
  • Amazon Web Services
    I use EC2, RDS, and Route 53. I chose not to use Elastic Beanstalk so that I would have to learn to SSH into a remote server and config it myself.
  • Bash
    For Angular CLI and Linux CLI on my EC2 instance.
  • WinSCP
    For transfering files to my EC2 instance.
  • WAMP PhpMyAdmin
    For both local and remote database management.

While this website probably doesn't look like much to the average viewer, I've tried to include some bells and whistles in the finer details.

Here are a few of the more notable aspects of this website.

Frontend

  • Adaptive navbar
    Standard top-bar for larger screens, and a drop-down menu for smaller screens.
  • Adaptive layout
    All pages should present well on a variety of screen sizes. For example, the resume page changes to vertical stacking on smaller screens, and the tooltips on this page are sized and positioned adaptively.
  • Browser compatibility
    Tested on Chrome, Firefox, Edge, Chrome for Android, and Firefox for Android. I hope to test on more platforms in the future but don't currently have the means.
  • Adaptive functionality
    Email nav button coppies to clipboard if possible, else displays it for the user to copy.

Breakout Game

  • Element creation and manipulation
    Generating elements, styling them, moving them, destroying them, etc.
  • User input
    Mouse, keyboard, and touch input for the paddle.
  • Collision detection
    Trickier than you might think due to corner clipping (i.e. tunneling).

High Scores

  • Custom cross-browser scrollbar
    See scores on home page. Works on all tested browsers. I sneakily hide the real scrollbar and semi-mimic it's properties.
  • Highlight new high scores
    When a new high score is entered at the end of the Breakout game, the relevant page of scores is retreaved, and then the new score is highlighted and scrolled to.

Backend

  • Semi-complex SQL queries
    The unordered nature of a relational database means that treating it as a ranked table of high-scores requires some fakery. A score's rank is generated as part of a query using table aliases, user variables, and table joins.
  • Data validation
    Prevent SQL injection.
